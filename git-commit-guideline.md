# Git Commit Guideline

## Git Commit 規範

- feat: 新的功能
- fix: 問題修正
- docs: 文件纇
- style: 格式變動（不影響城市功能）
- refactor: 重構，不影響現有功能也不新增功能
- perf: 效能優化
- test: 新增遺漏掉的測試或修正已存在的測試
- chore: 與環境建立、套件安裝修改

### Example

```
feat: 會員登入功能
fix: 修改會員登入過程遇到無法跳轉的問題
chore: 安裝 barryvdh/laravel-ide-helper 套件
```
